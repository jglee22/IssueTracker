# Issue Tracker 프로젝트 기능 설명

## 📋 프로젝트 개요

이슈 트래커는 프로젝트의 버그, 기능 요청, 작업 항목을 체계적으로 관리하는 풀스택 웹 애플리케이션입니다.

---

## 🔐 1. 사용자 인증 및 권한 관리

### 기본 인증
- ✅ 회원가입 (이메일, 사용자명, 비밀번호)
- ✅ 로그인 (JWT 토큰 기반)
- ✅ 비밀번호 암호화 (bcrypt)
- ✅ 이메일 중복 방지

### 승인 기반 회원가입 시스템
- ✅ 관리자 승인 필요 (PENDING → ACTIVE)
- ✅ 회원가입 시 PENDING 상태로 대기
- ✅ 관리자가 승인/거부 처리
- ✅ 거부 시 사유 입력 가능
- ✅ 첫 번째 사용자는 자동으로 ADMIN 권한 부여

### 역할 기반 접근 제어 (RBAC)
- ✅ **ADMIN**: 모든 권한 (사용자 관리, 프로젝트 관리)
- ✅ **MEMBER**: 일반 사용자 권한
- ✅ **PENDING**: 승인 대기 중 (로그인 불가)
- ✅ **REJECTED**: 거부됨 (로그인 불가)

---

## 👥 2. 관리자 기능

### 사용자 관리 페이지
- ✅ 가입 신청자 목록 조회 (PENDING 상태)
- ✅ 활성 사용자 목록 조회 (ACTIVE 상태)
- ✅ 거부된 사용자 목록 조회 (REJECTED 상태)
- ✅ 사용자 승인 (역할 지정 가능: ADMIN/MEMBER)
- ✅ 사용자 거부 (사유 입력 가능)
- ✅ 사용자 상태별 필터링

---

## 📁 3. 프로젝트 관리

### 프로젝트 CRUD
- ✅ 프로젝트 생성 (이름, 설명)
- ✅ 프로젝트 수정
- ✅ 프로젝트 삭제
- ✅ 프로젝트 목록 조회
- ✅ 프로젝트 상세 정보 조회

### 프로젝트 멤버 관리
- ✅ 멤버 추가 (이메일로 검색)
- ✅ 멤버 역할 설정 (OWNER, MEMBER, VIEWER)
- ✅ 멤버 역할 변경
- ✅ 멤버 제거
- ✅ 멤버 목록 조회

### 프로젝트 권한 시스템
- ✅ **OWNER**: 프로젝트 소유자 (모든 권한)
- ✅ **MEMBER**: 멤버 (이슈/댓글 작성/수정 가능)
- ✅ **VIEWER**: 조회 전용 (읽기만 가능)

### 프로젝트 대시보드
- ✅ 이슈 상태별 통계 (Pie Chart)
- ✅ 우선순위별 통계 (Bar Chart)
- ✅ 담당자별 이슈 수
- ✅ 최근 업데이트된 이슈 목록

## 🎯 4. 이슈 관리

### 이슈 CRUD
- ✅ 이슈 생성 (제목, 설명, 우선순위, 담당자, 라벨)
- ✅ 이슈 수정
- ✅ 이슈 삭제
- ✅ 이슈 상세 조회
- ✅ 이슈 목록 조회

### 이슈 상태 관리
- ✅ **OPEN**: 열림
- ✅ **IN_PROGRESS**: 진행 중
- ✅ **RESOLVED**: 해결됨
- ✅ **CLOSED**: 닫힘
- ✅ 드래그 앤 드롭으로 상태 변경 (Kanban 보드)

### 이슈 우선순위
- ✅ **LOW**: 낮음
- ✅ **MEDIUM**: 보통
- ✅ **HIGH**: 높음
- ✅ **URGENT**: 긴급

### 이슈 필터링 및 검색
- ✅ 프로젝트별 필터링
- ✅ 상태별 필터링
- ✅ 우선순위별 필터링
- ✅ 담당자별 필터링
- ✅ 제목/설명 검색 (키워드)

### Kanban 보드
- ✅ 드래그 앤 드롭으로 이슈 상태 변경
- ✅ 상태별 컬럼 표시
- ✅ 실시간 업데이트 (SSE)

## 🛠️ 기술 스택

### Backend
- Node.js + Express + TypeScript
- PostgreSQL + Prisma ORM
- JWT 인증
- Multer (파일 업로드)
- Server-Sent Events (SSE)

### Frontend
- React + TypeScript + Vite
- React Query (데이터 페칭)
- React Router (라우팅)
- Tailwind CSS (스타일링)
- Recharts (차트)
- React Hot Toast (알림)
- React DnD (드래그 앤 드롭)
